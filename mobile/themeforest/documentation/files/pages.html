Media - Galleries

Galleries are easy to build. They require however a precise class structure to work. Let's explain the absolute basic gallery structure. This is vital and must be followed

<div class="gallery">
    <a class="show-gallery" href="images_location/image.jpg" title="Image Caption When Opened">
        <img src="images/empty.png" data-src="image_location/image.jpg" class="preload-image" alt="img">
    </a>
</div>

 

The above structure will make an image open in full screen. There are a few required steps to be followed. 

Line 1 creates the gallery housing tag. This is mandatory as this will tag the gallery in JS

Line 2 creates the anchor of the image that gets tapped. This contains the mandatory show-gallery class that will allow the image to be opened, the href that contains the image that will be shown and the tile of the gallery that will appear on top of the opened image

Line 3 is the actual image itself. This is preloaded as explained in the features/images chapter.

Line 4 closes this image. 

If you wish to create more gallery elements simply duplicate lines 2 3 4 and replace the images with your own

Line 5 closes the entire gallery.

This is the main structure that will be used in all our galleries. Everything else will be just CSS styleing and secondary classes. We'll explain as needed.

Articles

Instant articles are made to simulate the design of Facebook. Please note! Facebook uses a huge server, that preloads stuff instantly when you open the app. We're usign HTML, meaning all the content gets loaded when you are on the page. If you plan on using LOTS of articles, it is recommended you split them into max 5 per page and use pagination. 

 

Instant Article Structure.

<div data-article-card="article-1" class="article-card bg-white">
    <a data-deploy-card="article-1" href="#" class="article-header">
        <span class="article-overlay"></span>
        <span class="article-image preload-image" data-src="images/pictures/8.jpg"></span>
        <span class="article-category color-white bg-border-light uppercase">Computers</span>

    </a>
    <div class="article-content">
        <h1 class="article-title top-15"> Article Caption</h1>
        <p class="half-bottom"> Text under the photo. </p>
    </div>
    <div class="article-content-hidden">
       <!-- This content willbe shown when opening the instant article ---> 
       <a href="#" class="close-article">Back to Articles</a>
    </div>
</div>

Lets explain

Line 1 creates the main article body. This has the article data-article-card="article-1"

Line 2 creates the article elemetns. This has the attribute data-deploy-card="article-1" 

Note that line 1 and 2 must have the same value for the data attribute

Line 3 is the article overlay

Line 4 is the article image. 

Line 5 is the article author. 

Line 6 is the category. 

Line 7 is the article time. 

Lines 4, 5, 6, 7 will be automatically placed in the main article when opened as well

Line 8 closes the article header.

Line 9 10 11 12 is the article content this will be visible in the opened article as well as the clickable one

Line 13 - 14 - 15 - 16 will be the article content. Copy and paste elements from UltraMobile features inside here.

Line 17 closes the instant article.

To make more, duplicate everything from line 1 to 17 and replace lines 1 and 2 data attributes with unique names.

Task Progress

Task Progress uses a progress bar system made by us. The typical progress bar plugin weighs over 30 kilobytes, and when our entire framework js weighs only 200 we decided it's absurd that such a small feature to weigh 15% of an entire framework.

Threfeore, we made this plugin. 

Before we begin. If you notice the page-task-progress.html page you'll see inside that each box shown there is a content-box as explained in the features page that uses different background colors and images with secondary classes. At the bottom of each of these is a progress bar. Let's explain how to build a progress bar.

<div class="progress-bar-wrapper"
     data-progress-complete="20"
     data-progress-height="23" 
     data-progress-round="true"
     data-progress-border="0" 
     data-progress-text-visible="true"
     data-progress-text-position="right"
     data-progress-text-color="#000000"
     data-progress-text-size="11"
     data-progress-text-before=""
     data-progress-text-after=" - Of Setup Complete"
     data-progress-bar-color="#3498db"
     data-progress-bar-background="rgba(0,0,0,0.2)">
     <div class="progress-bar"></div>
</div>

Line 1 creates the progress bar

Line 2 is the percentage of completion ( from 0 to 100

Line 3 is the height of the progress bar

Line 4 is the border, true is round, false is square

Line 5 is how thick the progress bar border should be

Line 6 is true to show text, false to just show the bar

Line 7 is the location of the text. Can be center, left or right

Line 8 is the color of the text.

Line 9 is the font size of the text inside the bar

Line 10 is the text before the percentage is shown

Line 11 is the text after the percentage is shown

Line 12 is the color of the progress bar that expands ( here are some great colors http://color.koya.io/ ) 

Line 13 is the color of the background of the progress bar ( here are some great colors http://color.koya.io/ ) 

Line 14 is the bar

Line 15 closes the progress.

You can create however many progress bars you want. Just as long as you duplicate all the lines above.

Check out http://color.koya.io/ and http://flatuicolors.com/ for great color combinations.

Covers - Important

Covers, are the elements that build coverpage sliders, center elements on a single screen, or simply create walkthroughs. These are vital to the function of any element and using the power of secondary classes, allow incredibly HUGE amounts of variations. 

Therefore, let's make a short introduction into cover-items the building element of any coverpage, single page pageapp, walkthrough or homepage cover slider and let's explain how each works. 

 

FAQ: What pages use the cover item? 

Pages like pageapp-error, pageapp-login, pageapp-login-2, pageapp-soon, pageapp-splash ( all of them ) pageapp-cover and pageapp-walktrhoughs ( all of them ) and also index-cover news-cover, store cover all use the same image slider, with different background images and our imagination when combining the secondary classes for colors. 

FAQ: What is the role of a cover-item?

A cover item will measure the size of your screen, width and height, and expand that element to that size. Meaning, it will be full screen. The list above should show you exactly their role and how flexibile they are.

FAQ: Can I add a tone of content and make the cover item scrollable?

No. Cover items cannot scroll. Please use limited content. These replicate the design on mobile devices, those do not scroll either. 

 

Cover Item Structure

Let me show you an example, this will make it much easier for you to understand how it works. Remember the 404 page we discussed a bit back? That has a content inside and nothing else on the page. Great! Well, if you copy that content inside a cover item, that will become centered on the page, and you can add a background behind it! 

<div class="cover-item" style="background-image:url(images/pictures_vertical/bg1.jpg);">

Coverpage FAQ's 

How many images / slides can I use in a cover slider?

As many as you want. There is no limit

Can I not use a background?

Yes! You can see examples of that in the pageapp-walkthroughs.

Can I use a video background?

No, that is a device limited feature.  Sliders and videos on mobiles will result into nasty crashes.

Can I add arrows to the coverpage slider?

That wouldn't be a great idea, but if you want, let us know and we'll explain how. Your users can only swipe left, right, up or down, and when the entire screen is an image that also has a timeout and dots at the bottom, they will pretty much figure it out as most mobile apps work on the same principle.

 

 

 

 

 

 

Walkthroughs

We placed walkthroughs right after the covers and coverpage sliders for a very important reason! They are one and the same thing! The coverpage slider and the walkthrough sldier work exactly on the same principle. The main difference is in the wrapping class. Let's explain.

 

Coverpage sliders use this wrapper class as explained in the previous chapter

<div class="cover-slider owl-carousel owl-has-dots-over">
      <!--your cover items go here-->
</div>
 
Walkthrough sliders however, allow you to add buttons for next and skip and also a main button at the bottom. Other than that you can customize each slide using elements from our template. 
 

Walkthrough with main button at the bottom.

The walkthrough with the main button on the bottom is pageapp-cover-walk-2.html and it has this structure

<div class="coverpage coverpage-full walkthrough-buttons walkthrough-has-button">
    <a href="#" class="next-slide-custom">Next</a>
    <a href="#" class="skip-slide-custom back-button">Skip</a>
    <a href="#" class="get-started-custom back-button button button-full button-blue uppercase ultrabold">Get Started</a>
    <div class="cover-walkthrough-slider owl-carousel">
       <!-- Cover items go here -->
    </div>
</div>

Line 1 opens the coverpage slider for the walkthrough. It has the secondary classes walkthrough-buttons and walkthrough-has-button. These classes are necessary for the following step

Line 2 is the next slide button this will go to the next slide. 

Line 3 is the skip button. For demo purposes we used the "back-button" class to go back where you came from in the demo

Line 4 is the get started main button at the bottom of the page. This is a fixed button. 

Line 5 defines this as a cover-walkthrough-slider (it has no auto play )

And inside line 6 you insert you cover slides. We used no backgrounds for ours and simple icons with some text, just like a regular walkthrough would look like.

Line 7 closes the cover-walkthrough-slider 

Line 8 closes the entire coverpage wrapper.
