Accordions

Style 1 - Classic Text with Arrow Down

<div class="accordion accordion-style-1 ">
    <a href="#" class="color-black font-14" data-accordion="accordion-1">Text Accordion<i class="fa fa-angle-down"></i></a>
    <div class="accordion-content" id="accordion-1">
        <p>This is the accordion content. You can add any content you want to it. Really, anything!</p>
    </div>
</div>

Line 1 defines the accordion and it's current style.

Line 2 is the activation of the accordion. Notice this has the attribute data-accordion="accordion-1", this same attribute will be used to identify which box needs to be showed. This identifier must be unique and only included inside the main accordion class.

Line 3 is the accordion content. This has the id accordion-1 which is directly conencted to the attribute on line 2. Line 2 will open this identifier. 

Line 4 closes the accordion content

Line 5 closes the accordion.

In order to create multiple accordions, simply duplicate lines 2 until 4 and replace the data-accordion and id found on line 2 and 3 with unique names, for example, if you create a new one, data-accordion="accordion-2" and id="accordion-2" and so on. 

Style 2 - Classic Text with Colors.

    <div class="accordion-content" id="accordion-1a">
        <p>This is the accordion content. You can add any content you want to it. Really, anything!</p>
    </div>
</div>

Line 1 Defines the accordion and it's current style

Line 2 defines the color and background color of the accordion button. Notice that it has the secondary classes for color and bg. Notice this has the attribute data-accordion="accordion-1a", this same attribute will be used to identify which box needs to be showed. This identifier must be unique and only included inside the main accordion class.

Line 3 is the accordion content. This has the id accordion-1a which is directly conencted to the attribute on line 2. Line 2 will open this identifier. 

Line 4 closes the accordion content

Background Infinite

Infinite backgrounds are great for spicing up pages. These must use a pattern image that repeats itself. 

This feature is only available for cover-item pages, and that is for a simple reason. This is a big element in the background of your device that will be constantly moving. For an iPhone or a Galaxy S device this is not a hard task to achieve, but for lower performing devices, it will be a big problem if a lot of content is added, therefore, we recommend you use this ONLY for the following pages

  • login screens
  • 404 page
  • coming soon page
  • pages with a simple modal
  • pages with no scroll
  • splash pages

Using it for complex elements will and can cause older devices to crash. This is a website in the end and we cannot acces native features to optimize it further than this.

<div class="cover-item">
    <div class="cover-content cover-content-center">
        <div class="content content-boxed content-boxed-padding">
            <!-- Insert Your Content Here -->
        </div>
    </div>
    <div class="cover-infinite-background"></div>
    <div class="cover-overlay overlay bg-black opacity-60"></div>
</div>

The code above must be alone on the page. You can only add it after the page-content-scroll class, and nothing after it.

Line 1 creates the cover item.

Line 2 makes the item centered on screen.

Line 3 creates the boxxed design and white background

Line 4 is where you add your content

Line 5 closes your content

Line 6 closes the content box design.

Line 7 is the infintite background image itself. You can locate this in framework.css replace there with your image.

Line 8 is the background effect over the image. It's set to black and 60% transparency.

Charts

The charts pages are very, very complex, they are easy to use and they can easily be customized. First, let's explain a few details about them

 

The charts.js script that runs this script is an absolute size monster, it weighs 100+kb, so in order to lighten the load on the page, we import it only on pages that actually run charts. In order to activate charts on a page you must add the class has-charts to the body of that page.

<body class="has-charts">

Be careful to only add it to the page you have charts on.

The second and most important aspect about charts is their load onto the page. The charts code is found in custom.js, simply use CTRL+ F for CMD + F to locate //Charts and you'll see the code there. 

Detailed tutorials about charts and exactly how and what you can do with them can be found on chart's js main plugin page. You can find exact exampels and full working charts there - > http://www.chartjs.org/

 

Please note! In both features-charts.html and features-graphs.html you MUST insert all chart classes you have created in custom.js, even if you do not use them, and inset the class "disabled" to disable it. Due to the fact that these elements are canvas, they will NOT load unless they are all present and disabled. This does not affect loading times.

Images + Preloading Images

Preloading Images

We use LazyLoad on our images to optimize the loading time of the page. Unlike most of our competitiors, we strongly believe that mobile pages should be blazing fast, and have caching and preloading throught them to provide a rich and fast experience.

Lazy loading an image is INCREDIBLY easy. Let's show you an example.

<img src="images.png" alt="img">

This is a regular image. Not preloaded, just the classic img class. 

<img class="preload-image" src="images/empty.png" data-src="image.png" alt="img">

And this is a preloaded image. Let's explain

class="preload-image" - tells the page this will be preloaded

src="images/empty.png" - points to an empty 1x1 pixel image in our images folder, this will load instantnly.

data-src="images/image.png" - is the actual location of the image. This image will be preloaded on your screen! 

And that's that! Use this for all images except those that are located in image sliders. We'll explain those after this chapter. 

 

Make images full screen, and maintain aspect ratio.

This is called responsive images, they will expand to fill the screen but will maintin aspect ratio. This is done by adding the class="responsive-image" to any image you have on screen. We use this almost everywhere in our template

Polaroid Effect

Polaroid Effects are awesome, you can do them using one of the following styles.

<img class="preload-image responsive-image polaroid-effect" src="images/empty.png" data-src="image.png" alt="img">

Firstly, we add the preload classes and responsive image classes, then we add the class polaroid-effect. This will create a white background with a 5 pixel padding around the image. If you wish to add text to the polaroid image, then, simply do as follows.

<span class="polaroid-effect">
    <img class="preload-image responsive-image" src="images/empty.png" data-src="image.png" alt="img">
    <em class="INSERT_SECONDARY_CLASSES_FOR_TEXT">Hello there!</em>
</span>

As you can see, we added the polaroid effect outside the image, in a separate wrapper. This will allow us to add a text under or above the image. You can use any secondary class for the text, from colors, to font sizes to opacity.

 

 

 

 

 

Image Sliders

There are a few image sliders made throughout the site, that are explained for each page they belong to, yet, it's important you also understand that the carousels can literally, slide anything. Any content you add in them wil work as 

Sliders can slide text and images alike without issues. 

 

First, let's explain the options you can use for your sldier. I'll create a demo class below, and we'll use that for our example. You can check examples in features-image-sliders.html

$('.demo-slider').owlCarousel({
    loop:true, 
    margin:0, 
    nav:false, 
    autoHeight:true, 
    lazyLoad:true, 
    items:1, 
    autoplay: true, 
    autoplayTimeout:3500
});

The above code must be added in custom.js, simply open custom.js and CTRL+F or CMD + F the "Owl Carousel Sliders" and you'll find the right place to put the code in. 

Line 1 determines the class that will house the image slider.

Line 2 determines if the sldier have a loop or not. Replace with false if you don't want to loop

Line 3 will determin the margin between slides (when you currently drag images, there is no space between them, replace with numbers)

Line 4 is the nav. Please leave this to false. We have classes that do this for you and will be explained shortly.

Line 5 is the autoHeight parameter. This is useful if you plan on having unequal slides in height, but please note, when this is activated, the entire page under the slider will animate as well. For this reason it's recommended to keep slides the same height.

Line 6 is the lazy loading of images inside the slider. I'll explain below how this is done. It's not the same as preloading classic images

Line 7 is the number of items you want to see per slide.

Line 8 is the autoplay function. Set it to false if you don't want it enabled

Line 9 is the timeout between slides. This is currently set to 3.5 seconds.

 

NOTE! 

The image slider classes on line 1 can be named however you wish, and in our slider, you can replace them wherever a slider appears to create more options. The only sliders who's classes MUST NOT be replaced are the cover, walkthrough and any fullscreen slider. These have special functions to enable them to be full screen. 

Let's say for example, you see on a page the class single-slider, and you want to create your own slider. Copy paste the JS code above inside the custom.js, modify it's options, and rename the class on line one, from .demo-slider to let's say .awesome-customer-slider. Then, in HTML, replace single-slider with awesome-customer-slider and the options will be changed.

This is very useful in case you want different timeouts for different sliders.

 

Creating a Basic Slider.

<div class="content">
    <div class="single-slider owl-carousel owl-no-dots">
        <div class="slider-item">
            <img class="owl-lazy responsive-image" data-src="images/pictures/8.jpg">
        </div> 
        <div class="slider-item">
            <img class="owl-lazy responsive-image" data-src="images/pictures/7.jpg">
        </div> 
    </div>
    (1) <!-- Slider Arrows Get Added Here -->
</div>

It's recommended that you place sliders inside a div class, wrapped around the entire element. If you want the sldier to have left and right margins, add it inside the content class, if you want the slider to extend full width, add it inside the container class.

Line 1 - the housing class of the entire slider. This must contain the slider and it's arrows. Nothing more

Line 2 - is the actual slider itself. This has the control class single-slider which I previously mentioned how you can update, the class owl-carousel which is targeted by CSS and JS and the secondary class owl-no-dots. You can replace owl-no-dots with owl-dots-under or owl-dots-above for ANY SLIDER we have shown. This will create the small control dots.

Line 3 is a individual slider item. In this case, we named it that way so you can easily see it, but for each slider this class can be different, some sliders like the coverpage or home sliders use special classes. 

Line 4 is the image inside the slider, you can also add text here as well. The image is loaded basically in the same fashion as a preloaded image, but, the main difference is 

- the slider does not need an empty src tag

- the slider preload class is owl-lazy NOT preload-image.

These elements combined will tell the JS to pre-fetch the next image right before the slider is finished loading, allowing an unintrerupted experience, when, in fact, images are loaded as they are required. 

Line 5 closes the individual slide item.

To create more slider elements, simply duplicate lines 3-4-5 and be sure that if they have other classes than the ones mentioned in line 3, you do not delete them.  Those classes may be vital to the positioning of the slider on screen. 

 

In the code above, you'll see a  (1) - Slider Arrows get added here. Where that code is you can add arrows for your slider. Be sure you have the slider inside a container or content class before doing so and be sure there is no other slider together with it in there. We use the parent location to determine where the slider is so the buttons target it. This is why we mentioned not using the nav parameter in the JS. Doing so would mean each individual slider would have CSS settings for the arrows, making the code much heavier, for no reason. 

Adding Slider Arrows

<a href="#" class="next-slide-arrow font-20 color-white"><i class="fa fa-angle-right"></i></a>
<a href="#" class="prev-slide-arrow font-20 color-white"><i class="fa fa-angle-left"></i></a>

The code above will generate arrows for the slider you're placing them in. Be sure they are inside the content/container class of the slider, not withing the slider itself. 

Adding Next Pre Buttons

<a href="#" class="next-slide-text color-black">Next <i class="icon-clear-left fa fa-angle-right font-14"></i></a>
<a href="#" class="prev-slide-text color-black"><i class="icon-clear-right fa fa-angle-left font-14"></i> Prev</a>
The code above will generate text arrows for the slider. This is great if you want the dots and arrows to appear but not overlay on the image. Check out the last example in the features-image-slider.html page.

OS Detection

You can detect if a user is browsing from Android or iOS and show content only for that user. This is great, if for example your website has an app that works for iOS and you want to show only the iOS button.

Search Box

Search is not connected to a database, but, it searches within a list you can generate using data lists you can add to any element 

<div class="content search-input-container search-input-round search-input-dark">

        <div class="search">
            <i class="fa fa-search"></i>

        <div class="search-results search-icon-list disabled-search-list">
            <div data-filter-item data-filter-name="all smartphone apple iphone"><i class="fa fa-apple"></i>Apple iPhone</div> 
            <div data-filter-item data-filter-name="all smartphone apple ipad"><i class="fa fa-apple"></i>Apple iPad</div> 
            <div data-filter-item data-filter-name="all smartphone htc one"><i class="fa fa-android"></i>HTC One</div> 
            <div data-filter-item data-filter-name="all smartphone google pixel"><i class="fa fa-android"></i>Google Pixel</div> 
        </div>
    </div>
</div>

Line 1 creates the content box that will house the search. You can remove the content class if you wish but the other classes must stay there. 

You can replace search-input-round with search-input-square and also search-input-light with search-input-dark.

Line 3 creates the search field

Now, notice each search result from line 8 9 10 and 11. Each has a data-filter-name attribute with some words inside. Those words are the keywords that your search will reply to. Inside each div you can add your own content. Or make the divs anchors when someone clicks them to be taken to a page.

Reducing text contrast

As you can see our page is light with dark text on it. You have the ability to change the text color using the COLOR-yourcolor secondary class mentioned in the structure pages, but what about if the color is too dark? 

We created a few opacity classes, thes are applicable to all elements on the page. When you add this class, the item you add it too will become more transparent. In the case of text, this will become less "proeminent" and more dimmed on screen. Use the following classes to achieve this 

PS: This is a great tool when using decorations that you want to look a bit dimmer.
