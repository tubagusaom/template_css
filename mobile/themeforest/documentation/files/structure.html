F.A.Q

What are those fa- things I keep seeing all over the item?

That is font awesome. You can find all available icons you can use on http://fontawesome.io/icons/

How do I change the contact form information?

The contact form is easy to edit. Open the php folder and the only php file there, edit it and replace mail@domain.com with your address. Please note! We've been using this exact same php for all our items, so if you don't recieve an email, please contact your hosting provider and request they enable all necessary php functions. Our contact form has been tested by over 20.000 customers and the only times an issue appeared it was due to the hosting company.

I want to make the page even faster, how do I do that?

First, create the page as you need it. Then, let's say you don't use the news, store and blog sections. Open framework.css and remove those elements completely by using CTRL + F or CMD + F to find the news, store, blog chapters and then simply delete them.

I don't want to use AJAX. How do I remove it? 

Open custom.js and delete the last function on the page. It should be around line 739 just under the setTimeout(init_template,0);. Remvoe everything except for the very last }); on the page. 

How do I change the settings to an image slider?

All image sliders on our page are are found in custom.js use ctrl+f or cmd+f to find //Owl Carousel Sliders. Add yours inside the settimeout function. We are usign owlCarousel 2.2.1 ( https://owlcarousel2.github.io/OwlCarousel2/docs/started-welcome.html )

How can I convert this to a WordPress Theme or PhoneGap / Cordova Application?

By default our item is HTML Site template, but it uses basic, simple html and css that is valid for both WordPress and PhoneGap. you can convert this to both of them by requesting the help of a freelance over at https://studio.envato.com/

 

Secondary Classes

To better understand the incredible flexibility our item uses, we'll use certian terms along the way, here is where we define those terms. This page is for all users, regardless of how proficient or experienced they are. We used simple terms in the idea that not all our customers will be pro's, and we believe everyone should be able to use our product.

 

Secondary Classes

The secondary classes are something that make our item much more flexbile, and lighter in weight than anything our competition has provided. A secondary class to a main class. Can be used multiple times in a div with different properties. For example, changing colors, font sizes, margins, etc.

<div class="main-class secondary-class-1 secondary-class-2 secondar-class-3"></div>

 

But why do you use them? Why not hard code colors and sizes and fonts into css?! Why do I choose?

It's simple. UltraMobile uses a lot of features, and like yourself, we want some of those features to look in a way, maybe you want them to look differently.

An instant example. We have a lot of button variations, used in different places with different styles. A button takes about 5 lines of code in CSS. Which doesn't ammount to much space. But, when you have over 100 pages, with a lot of button styles and different colors, adding just 5 lines of code to each button would result in over 500 lines of code in CSS. Making the item rigid, incredibly hard to customize, and increasing it's weight tenfold. Having reusable classes allows us to create a class, like a color for example, that can be applied in the HTML however many times you want, without increasing the page size and decreasing the load speed. 

But secondary classes usually have ugly names and are hard to understand!

Not at all. We used basic english for each class name. Below, you'll see all of them, explained carefuly. Throught the documentation we'll mention exact places where you can suse them with great ease and where they trully rock!

 

Control Classes

disabled - will display:none ( hide ) the element you add this class to. Great use in header / menu elements or when you beta test an element and want to remove it from screen without having to delete it. It's great for learning.

close-menu -  adding this class ot any item of any menu will close that menu  when tapped. Applies to menus, walkthrough-boxes, action-sheets or any element that uses menus.

default-link - if you plan on using external links in the item, meaning, any link that uses www  or http or https or anything external you will have to add this class .This class will bypass the AJAX activation and allow accesing external links. Otherwise, you'll get an endless loader icon on the screen.

 

Typography Secondary Classes

These can be added to text elemements to change their colors, sizes and font weights. Say for example you want a heading to be red, but you need it on multiple pages, in different sizes. Rather than creating a tone of CSS for that, you just add the following secondary classes.

 

center-text - aligns text to center

right-text - aligns text to right

left-text - aligns text to left

---

capitalize - will capitalize your font, Making It Look Like This

uppercase -  will uppecase your font, MAKING IT LOOK LIKE THIS

lowercase -  will lowercase your font, making it look like this

thin - will make your font the thinest possible weight

thiner - will make your font a bit thiner than the default value.

regular - will make strong text look like normal text

regularbold - will make headings look less bold.

bold - will make text bold

ultrabold - will make text the boldest possible variation

---

small-text - will make the font-size of a text 12 pixels, and adjust line height

smaller-text -  will make the font-size of a text a bit smaller than the default one, and adjust line height

larger-text - will make your text 26 pixels ( a bit larger than the h1 tag )

large-text -  will make your text 28 pixels ( a bit more larger than the h1 tag )

huge-text - will make your text 30 pixels, very, very large.

---

boxed-text - will automatically center your text and limit it to 80% of the screen width

boxed-text-small - will automatically center your text and limit it to 200 px ( most screens are 320 - 340px )

boxed-text-large - will automatically center your text and limit it to 300 px ( most screens are 320 - 340px )

 

Icon size manipulation and text size manipulation.

Maybe our font sizes are perfect for you, but you need something that's right inbetween for a heading. Normally, you go to CSS and create that class. But there's no need. We created some font classes that will alow you to edit your text, but most importantly, these have been built to manipulate the size of font-awesome icons. These are placed in the header, footer, and other places, and being able to edit their sizes in just one place will be incredibly useful in the navigation and header designs.

Simply add the following classes to any text or icon element to alter it's size without having to edit any css. font-10, font-11, font-12, font-13, font-14, font-15, font-16, font-17, font-18, font-19, font-20, font-21.

For example, let's say you have a facebook icon inside a paragraph. By default, our font size for paragraphs is set to 13 pixels. You can increase both the size of the text and icon by adding the secondary class to the paragraph or simply adding it to the icon will change it's size alone.

<p><i class="fa fa-facebook font-15></i>The icon will be 15 pixels, i'm just 13 pixels</p>

 

Position & Column Secondary Classes

These classes are used to affect the margin bottom or top of an element or their overall position on the screen

top-VALUE / bottom-VALUE - you can replace value with:

0 - 0px margin top/bottom

5 - 5px margin top/bottom

10 - 10px margin top/bottom

15 - 15px margin top/bottom

20 - 20px margin top/bottom

25 - 25px margin top/bottom

30 - 35px margin top/bottom

In previous templates we used 15px increments, but , our customers requested we add even more granular control over the top and bottom margin, therefore, this version allows you a much finer top and bottom control.

These will affect the top or bottom margin depending on which you choose.

For example, a paragraph has the bottom margin of 30 pixels. Maybe you want that to be smaller or no existant at all in certain places. Maybe you want to give it a maximum top margin over another element. Then you would do this

<p class="bottom-0 top-30">I no longer have a bottom margins but I have 30pixels margin top</p>

 

Image Effects

polaroid-effect / polaroid-effect-small - will add a polaroid white padding to the image

responsive-image - will make the image be full width, while maintaing it's aspect ratio

preload-image - this class is added to images you want to preload

<img src="location_to_empty_image.png" data-src="location_to_image.png" alt="img" class="preload-image">

 

Color and Background Color Classes

Color classes can be used for text elements, but backgrounds can be used for most elements shown in the page.

Background secondary classes are - bg-COLOR_NAME-light / bg-COLOR_NAME-dark

Color secondary classes are - color-COLOR_NAME

Border secondary classes are - border-COLOR_NAME-light / border-COLOR_NAME-dark

The available colors to replace COLOR_NAME are : teal, green, green2, blue, blue2, magenta, night, yellow, yellow2, orange, red, pink, magenta, brown, brown2.

 

You can also easily manipulate the "intensity" of colors. For example, our template uses dark text on light background, mostly all over the item. So for example, let's say you have a paragraph that you decide to make pure black

<p class="color-black SECONDARY_CLASS">I'm a black colored text</p>

But you decide, that black is too dark for your white background and you want to alter it's contrast. Normally, you'd go in the CSS and alter the class or create a special class for it. Well, you don't have to. You can easily modify the intensity of the color by modifying it's opacity. Replace the SECONDARY_CLASS with the following classes, a larger number means more opaque, a smaller number means more transparent, the classes are: opacity-10, opacity-20, opacity-30, opacity-40, opacity-50, opacity-60, opacity-70, opacity-80, opacity-90.

This can also be aplied to other colors in the item, or to entire elemenets. Using for example opacity-90 will show a very dark shade of gray, but using opacity-50 will show a lighter one. If you want to have a small text that is bearly visible, then you simply use opacity-30 small-text. Simple, right?

 

Overlays.

Throughout our template you'll see overlay classes. These have the ability to darken or lighten the text behind them, but they can also change colors.

The default overlay classes are overlay-dark (80% black), overlay-gradient (top light, bottom dark ) and overlay-light ( 75% white), but, you can do some amazing things with reusable clases. Imagine you'd want a slider to have 5 different overlays, and you'd only have the overlays above, that would mean a lot of css, right? Correct! Using our method, you'll be done in no time!

Please note, the structure below is just for demo purposes. Use the overlays in their designeated places as shown thoghout the documentation

<div class="overlay overlay-dark">I am the default dark overlay, I'm 80% black opacity</div>
<div class="overlay bg-black opacity-50">I am a custom overlay, I'm black, with 50% opacity</div>
<div class="overlay bg-red-dark opacity-70">I am a custom overlay, I'm red with 70 opacity</div>

 

Now, you'll see we use these secondary classes in almost every location of the template. This is makes it easier for you to customize elements, give them your personal feel and look, or just use them as we left them and as the demo you love and purchased for exists.

We give you the demo design + the uncanny ability to customize every element!

Structure

UltraMobile has a complex yet easy structure, built around the idea of extreme flexibility, we created 3 main wrappers. Let's explain everything. 

Page Structure

<div id="page-transitions">
    <div id="header">
        <!-- More in Header Chapter-->
    </div> 
    <!--Menus will be added here-->
    <div class="page-content"> 
        <!-- Everything will be created here --> 
    </div>
</div>

The main housing class of the page is page-transitions this identified will be used to change the content of the page using AJAX. This is vital and should not be forgotten or removed. If you wish to remove the AJAX loader alltogether, please send us a message and we'll gladly send you over the exact lines of code that need to be removed.

Line 2,3,4 is the page header. This will hold the top icons that are always fixed.

Line 5 will house the menus. We'll get to those in a second

Line 6 is the page-content class. We use this for measurements when creating coverpages, sliders and other elements that require an easy to use element that's not the body

Line 7 is where you add all page content! WARNING! Everything you build except for menus, action sheets, notification boxes, toats MUST BE ADDED HERE.

Lines 9, 10 close the entire structure.

 

page-transitions  - is used to generate the page effects and create the ajax load effect

page-content - this are vital to the functioning of the template. These elements are used for measuring the sizes of the menus that will be added to your screen, and these are the elements that get animated when you tap to open any menu item. ALL CONTENT with the exception of menus, action-sheets, toasts, notification boxes MUST BE ADDEDED INSIDE THIS CLASS.

Containers & Content

Container Class

Containers and contents have very distinct and simple purposes. 

When creating elements that are dependant to one another, we suggest adding them inside the container class. This will asure there are no errors with the elements and also add a neet 30 pixel margin bottom to keep visuals elegant. 

<div class="container">
    <!-- Insert your stuff here -->
</div>

The container class has no other property except the margin-bottom. This is great to keep things in blocks and easy to see around the page. 

 

Content Class

The content class however, has a more important role. You'll see this alot throughout UltraMobile. The content class is used to make left and right paddings so your content doesn't stick to the end of your device. When the content class is not set, that element is full width, when you place a content class, everything inside it will automatically be spaced from the left and right side of the screen.

<div class="content">
    <!-- Content that is not full width will be here -->
</div>

Changing Header, SIdebar and Footer logos.

Changing the logos is simple. Due to the fact that UltraMobile has highlights, the logo we used in the demo depends on the color of the highlight. Let's explain.

In style.css at the top you will find all the highlights. Let's say we want to change the logo for highlight-red. Open style.css and at the very top locate this code

.highlight-red .header-light .header-logo{background-image:url(../images/logo-dark-red.png);}
.highlight-red .header-dark .header-logo{background-image:url(../images/logo-light-red.png);}
.highlight-red .footer-light .footer-logo{background-image:url(../images/logo-dark-red.png);}
.highlight-red .footer-dark .footer-logo{background-image:url(../images/logo-light-red.png);}
.highlight-red .menu-dark .sidebar-logo a{background-image:url(../images/logo-light-red.png);}
.highlight-red .menu-light .sidebar-logo a{background-image:url(../images/logo-dark-red.png);}

In the code above, replace the destination to the logo with the destination to your own logo.

Now, we need to set the sizes for the logo inside the header, sidebar and footer. In order to do that, in style.css at the very bottom you will see this code.

.sidebar-logo a{
    background-repeat: no-repeat;
    background-size:136px 16px;
    height:20px;
}

.header .header-logo, .footer .footer-logo{
    background-size:115px 13px;
    width:115px;
    margin-left:-57px;
}

.footer .footer-logo{
    background-size:175px 20px;
    position:relative;
    width:100%;
    height:20px;
    left:auto;
    margin:0 auto;
}

Replace the background-size of each parameter seen above with half the size of your logo. This will maintain a crisp, elegant retina look. IF needed, increase the hight parameter as well.

And that's it! That's how your replace the logos!

Header

From past experience we learned that customers like insane flexibility over the header. Some prefer a center logo, others a left one, some to the right, some to the left. We included all these styles, and editing them is absolutely easy.

Let's start with the absolute basic. The general structure of the header.

Header Sturcture

<div id="header" class="YOUR_HEADER_STYLE HEADER-COLOR"> <!-- Header Here --> </div>

This is your header. It has the ID header which we use to manipulate when moving the menu.

YOUR_HEADER_STYLE can be replaced with

header-logo-left - will place the logo on the left side of the header, 4 icons right

header-logo-right - will place the logo on the right side of the header, 4 icons left

header-logo-center - will place the logo in the center, 2 icons left, 2 icons right.

header-logo-app - will place 1 icon to the left, the header logo and 3 icons to the right.

HEADER-COLOR  - can be replaced with header-light, header-dark, or if you feel experimental, check out the background color secondary classes and have fun!

On pages where you use header cards, these get added inside the page-content class. You can hide the header and make it show again by adding the class hide-header-card to the header.

 

Header with Left, Center or Right  Logo

        <a href="#" class="header-icon header-icon-2"> <!--Icon Here--> </a>

Replace header-logo-value with header-logo-left, header-logo-center or header-logo-right to switch the style of the header.

Header with App Style design.

    <a href="#" class="header-icon header-icon-2"> <!--Icon Here--> </a>

In this style, you'll create the app like logo. The icon to the left, followed by the logo or page title, then another 3 icons. To choose between the header logo or a page title, simply remove the disabled secondary class from the one you want to show.

Menus

We're excited and proud to show off our menu system. UltraMobile uses the most complex and easy to use off-canvas system on the entire Envato Marketplace. No other menu can do anything even remotely close to what ours can do. What is that you say? Everything! You can deploy sidebars, header dropdowns, footer fly ups, modals, and even create off canvas custom elements with INSANE ease!

First, let's explain the method behind it.

We use data-attributes. This allows us to tag any menu from a data-class then target the identified of the box you want to animate. In simple language, you add a data-deploy-menu="your-menu-1" to any button, and then id="your-menu-1" to a menu box, and voila! Together with a few clases, that menu will activate.

Let's start off!

Creating any Menu

<a data-menu="menu-1" href="#">Deploy Menu</a>

The code above has the data-menu="menu-1". This is a unique name, be sure you use it only for a single menu style, for example, data-menu="menu-1" and id="menu-1" are the unique identifiers for our sidebar menu.

If you plan on creating another menu, you can used menu-2 or any unique name you want, including you-are-an-awesome-custoemer. As long as it matches the ID of the menu, it will open it.

Creating Sidebars

In UltraMobile we've created 5 different sidebar variations, each with a different effect. These are easy to implement and can be done by follwing this basic structure ( more in the next chapter on how to create menu items )

<div id="menu-1" class="menu-sidebar menu-COLOR menu-sidebar-POSITIOON menu-sidebar-TYPE">
    <div class="menu-scroll">
       <!--More in the next chapter-->
    </div>
</div>

Line 1 generates the menu. This has the id identical to the data-menu that you want to target.

menu-COLOR can be replaced with menu-dark or menu-light

menu-sidebar-POSITION can be replaced with menu-sidebar-left or menu-sidebar-right

menu-sidebar-TYPE can be replaced with menu-sidebar-3d, menu-sidebar-reveal, menu-sidebar-push, menu-sidebar-over or menu-sidebar-parallax.

NOTE: You can have any number of sidebars on each side but you CANNOT open a sidebar from a sidebar. The transitions required to generate these sidebars will cause errors. Sidebars can only be opened from the page-content or outside the page content, but not from another sidebar.

Line 2 is the menu-scroll. This is essential to allow your menu to scroll in case your content exceeds the screen height.

Creating Flyin Menus / Action Boxes

A good example of a flyin menu is the share button located in the footer. These are incredibly easy to do. You can target them exactly like you target the sidebars, using the data-menu attribute. And the data-menu must be identical to the ID of the menu

<div id="menu-1" class="menu-flyin">
    <div class="menu-flyin-content">

    </div>
</div>

Line 1 generates the menu. The class menu-flyin is necessary

Line 2 generates the internal content class. Add all your content here. Check the component-action-boxes.html page for copy paste examples of these menus.

Creating Menu Items

<div class="menu-items menu-icons">
    <a href="#"><i class="fa fa-star"></i>Welcome</a>
    <a href="#" class="menu-item-active"><i class="fa fa-star"></i>Homepages</a>
    <a href="#" data-submenu="submenu-1" class="submenu-has-numbers"><i class="fas fa-star"></i>Templates</a>
    <div class="submenu submenu-active" id="submenu-1">
        <a href="#">Store</a>
        <a href="#">News</a>
        <a href="#">Blog</a>
    </div>
</div>

The structure above generates the menu items. These are very easy to use. Lets explain

Line 1 creates the menu housing class. This can have the class menu-icons to show the first icon or menu-no-icons to hide the first icon.

Line 2 is a simple menu item. This is not activated.

Line 3 is a simple menu item. This is activeted by adding the class menu-item-active

Line 4 is a simple menu item. Notice that this has the data-attribute data-submenu="submenu-1" this will be used to target the submenu on line 5 that will have the id identical with the data-submenu, meaning, submenu-1. The submenu can also have a plus icon if needed. You can add it by replacing submenu-has-numbers with submenu-has-plus.

Line 5 is the submenu itself. This contains basic links for each submenu element. Adding submenu-active to this will make the submenu show as deployed when the page is loaded.
